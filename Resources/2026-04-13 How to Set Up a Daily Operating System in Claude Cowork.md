# How to Set Up a 'Daily Operating System' in Claude Cowork

Zwięzły przewodnik krok po kroku do ustawienia "Daily Operating System" w [[Claude Cowork]] — skrócona wersja praktycznego workflow opisanego szerzej przez JJ Englerta. Trzy kroki tworzą fundament projektu: folder na komputerze, plik `brain.md` z kontekstem osobistym i połączenie folderu z projektem Cowork przez aplikację desktopową. To minimalna, wystarczająca instrukcja do uruchomienia systemu od zera.

## Frameworki i metody

- **Projekt jako folder lokalny** — folder `Daily Operating System` na komputerze staje się trwałym workspace w [[Claude Cowork]]; fizyczna lokalizacja plików i pamięć projektu są ze sobą powiązane
- **Plik `brain.md`** — markdown z preferencjami pracy, listą współpracowników i stałymi instrukcjami; wczytywany automatycznie przy każdej sesji projektowej
- **Instrukcja projektu** — krótki prompt definiujący ogólny cel projektu, podawany przy tworzeniu projektu w aplikacji Cowork

## Wnioski

- Trzy kroki wystarczą do uruchomienia trwałego, kontekstowego workspace w [[Claude Cowork]] — próg wejścia jest bardzo niski i możliwy do wykonania w 10 minut
- Plik `brain.md` jest kluczowym elementem całej architektury — bez niego projekt działa jak zwykły czat bez pamięci między sesjami
- Ten wzorzec jest punktem wyjścia do rozbudowy o konektory, umiejętności i automatyzację — nie wymaga wiedzy technicznej
