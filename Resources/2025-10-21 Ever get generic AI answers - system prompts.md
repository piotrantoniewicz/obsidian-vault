# Ever get generic AI answers - system prompts

Newsletter wyjaśnia mechanizm system promptów w [[Alice]] — trwałego kontekstu konfigurującego asystenta raz, zamiast powtarzania go w każdej rozmowie. Autor podaje cztery elementy dobrego system promptu: tożsamość (kim jest asystent), ton (jak komunikuje), wiedza (co wie) i cel (czemu służy). Dobrze napisany system prompt przekształca generyczny model językowy w wyspecjalizowanego asystenta — np. marketera znającego ton marki czy mentora w stylu określonej osoby. To samo podejście leży u podstaw budowania asystentów AI dla organizacji.

## Wnioski

- Cztery elementy system promptu (tożsamość, ton, wiedza, cel) to praktyczny framework do budowania asystentów AI dla NGO — można go bezpośrednio stosować przy projektowaniu narzędzi dla organizacji (np. asystent do pisania wniosków, asystent fundraisingowy).
- System prompt eliminuje barierę „muszę za każdym razem tłumaczyć AI kto jestem" — kluczowy argument przy szkole AI dla organizacji, które rezygnują z AI przez brak czasu na kontekstualizowanie każdego pytania.
- [[Prompt engineering]] dla asystentów to nie jednorazowa konfiguracja, lecz iteracyjny proces — zasada „zacznij prosto, udoskonalaj w trakcie" jest dobra metodycznie i dobrze komunikuje się uczestnikom szkoleń.

## Cytat

> Jeden setup. Niekończące się mądre rozmowy — system prompty pozwalają zbudować asystenta, który już zna Twój kontekst.

## Zastosowanie

Framework czterech elementów system promptu (tożsamość, ton, wiedza, cel) można wprost przenieść do warsztatu szkoleniowego dla NGO: uczestnicy budują własnego asystenta organizacyjnego krok po kroku, od razu widząc różnicę w jakości odpowiedzi.
